# PostgreSQL Database Schema Documentation

This document outlines the PostgreSQL database schema for the modern e-commerce platform, including the Brand, Model, and updated Products tables.

## Tables Overview

### 1. Brand Table
Stores brand/manufacturer information for products.

```sql
CREATE TABLE public."Brand" (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  name text not null,
  image_url text null,
  constraint Brand_pkey primary key (id),
  constraint Brand_name_key unique (name)
) TABLESPACE pg_default;
```

**Columns:**
- `id` - Primary key (auto-generated bigint)
- `created_at` - Timestamp when brand was created (default: now)
- `name` - Brand name (unique, required)
- `image_url` - URL to brand logo/image (optional)

**Indexes:**
- `idx_brand_name` - On `name` column for faster lookups

---

### 2. Model Table
Stores product model information with specifications.

```sql
CREATE TABLE public.model (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  year integer null,
  name text null,
  specs text null,
  constraint model_pkey primary key (id),
  constraint model_name_key unique (name)
) TABLESPACE pg_default;
```

**Columns:**
- `id` - Primary key (auto-generated bigint)
- `created_at` - Timestamp when model was created (default: now)
- `year` - Model year (optional)
- `name` - Model name (unique, optional)
- `specs` - Model specifications (optional)

**Indexes:**
- `idx_model_name` - On `name` column for faster lookups

---

### 3. Products Table (Updated)
Represents products with enhanced attributes and relationships.

```sql
CREATE TABLE public.products (
  id uuid not null default gen_random_uuid (),
  title text not null,
  url text not null,
  category_name text not null,
  owner_id uuid null,
  price numeric(10, 2) not null,
  condition text not null,
  cover_image_url text not null,
  brand text null,
  model text null,
  item_specifics jsonb null,
  created_at timestamp without time zone null default now(),
  discount real null,
  stock smallint not null default '0'::smallint,
  part_number text null,
  constraint products_pkey primary key (id),
  constraint products_url_key unique (url),
  constraint products_brand_fkey foreign KEY (brand) references "Brand" (name),
  constraint products_category_name_fkey foreign KEY (category_name) references categories (name) on update CASCADE on delete RESTRICT,
  constraint products_model_fkey foreign KEY (model) references model (name),
  constraint products_owner_id_fkey foreign KEY (owner_id) references profiles (id) on delete CASCADE
) TABLESPACE pg_default;
```

**Columns:**
- `id` - Primary key (UUID, auto-generated)
- `title` - Product title (required)
- `url` - Product URL slug (unique, required)
- `category_name` - Product category (required, foreign key to categories)
- `owner_id` - Product owner/seller (foreign key to auth profiles)
- `price` - Product price (decimal: 10,2, required)
- `condition` - Product condition (required)
- `cover_image_url` - Main product image (required)
- `brand` - Brand name (optional, foreign key to Brand.name with CASCADE)
- `model` - Model name (optional, foreign key to Model.name)
- `item_specifics` - Additional item specifications as JSONB (optional)
- `created_at` - Timestamp when product was created
- `discount` - Discount value/percentage (optional)
- `stock` - Number of items in stock (default: 0)
- `part_number` - Product part/SKU number (optional)

**Indexes:**
- `idx_products_category_name` - On `category_name`
- `idx_products_owner_id` - On `owner_id`
- `idx_products_price` - On `price`
- `idx_products_brand` - On `brand`
- `idx_products_model` - On `model`
- `idx_products_part_number` - On `part_number`

**Foreign Keys:**
- `products_brand_fkey` → Brand(name) - ON UPDATE CASCADE, ON DELETE SET NULL
- `products_category_name_fkey` → categories(name) - ON UPDATE CASCADE, ON DELETE RESTRICT
- `products_model_fkey` → model(name) - ON UPDATE CASCADE, ON DELETE SET NULL
- `products_owner_id_fkey` → profiles(id) - ON DELETE CASCADE

## Relationships

```
Brand (1) ──→ (Many) Products
  - Products can reference 0 or 1 Brand
  - When Brand is deleted, products.brand is set to NULL
  - When Brand name is updated, products.brand is updated CASCADE

Model (1) ──→ (Many) Products
  - Products can reference 0 or 1 Model
  - When Model is deleted, products.model is set to NULL
  - When Model name is updated, products.model is updated CASCADE

Categories (1) ──→ (Many) Products
  - Products must reference 1 Category
  - Restricts deletion of categories if products exist

Profiles (1) ──→ (Many) Products
  - Products can reference 0 or 1 Profile (owner/seller)
  - When Profile is deleted, associated products are deleted CASCADE
```

## Usage Examples

### Find products by brand:
```sql
SELECT p.* FROM products p
WHERE p.brand = 'Toyota'
ORDER BY p.created_at DESC;
```

### Find products by model with specifications:
```sql
SELECT p.*, m.specs FROM products p
LEFT JOIN model m ON p.model = m.name
WHERE p.brand = 'Honda' AND m.year = 2023;
```

### Get item specifics:
```sql
SELECT p.title, p.item_specifics
FROM products p
WHERE p.item_specifics IS NOT NULL;
```

### Count products by brand:
```sql
SELECT p.brand, COUNT(*) as count
FROM products p
GROUP BY p.brand
ORDER BY count DESC;
```

## Best Practices

1. **Brand Management**
   - Always use lowercase or consistent casing for brand names
   - Validate unique constraints before inserting

2. **Model Information**
   - Use JSONB for complex specs instead of storing as text
   - Consider normalizing frequently queried specs

3. **Product Updates**
   - When importing products, validate brand and model exist
   - Use CASCADE on brand/model updates to maintain consistency

4. **Performance**
   - Ensure indexes are created on frequently filtered columns
   - Consider partitioning products table if it grows beyond millions of rows

5. **Data Integrity**
   - Brands cannot be deleted if products reference them (will set to NULL)
   - Categories cannot be deleted if products reference them (RESTRICT)
   - Profiles/owners should be deleted with CASCADE for data cleanup
